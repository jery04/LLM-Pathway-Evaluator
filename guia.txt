1. ¿Qué significa “resolver un problema de optimización, planificación, satisfacción de restricciones…”?

Eso se refiere a técnicas típicas de IA clásica:

Optimización → buscar la mejor solución entre muchas posibles.
Planificación → construir una secuencia de pasos válida para lograr una meta.
Satisfacción de restricciones (CSP) → encontrar soluciones que cumplan reglas o restricciones.

Ejemplos:

Optimización

Encontrar la mejor trayectoria profesional según:

salario máximo
menor tiempo de estudio
mayor estabilidad
compatibilidad con habilidades del usuario

Ejemplo:

Opciones:

Frontend → Full Stack → Tech Lead

vs

Data Analyst → Data Scientist → ML Engineer

El sistema puntúa y optimiza.

Planificación

Construir un plan ordenado.

Ejemplo:

Meta:

“Quiero ser Machine Learning Engineer”

El sistema crea un camino:

1. Aprender Python
2. Aprender estadística
3. Aprender ML
4. Hacer proyectos
5. Conseguir internship
6. ML Engineer Junior

Aquí hay dependencias.

No puedes hacer:

Deep Learning

sin antes:

Python + álgebra lineal
Restricciones (CSP)

Buscar trayectorias válidas bajo reglas.

Ejemplo:

Usuario:

Tiempo máximo: 2 años
Presupuesto: bajo
No quiero matemáticas avanzadas
Ya sé programación
Quiero trabajar remoto

Entonces el sistema debe filtrar trayectorias inválidas.

No podría recomendar:

Computer Vision Research Scientist

porque viola restricciones.

No basta con inventarlas.

Deben cumplir criterios.

Ejemplo:

Criterio 1 — rapidez

La trayectoria minimiza tiempo.

Criterio 2 — salario

Maximiza ingresos.

Criterio 3 — afinidad

Maximiza coincidencia con intereses.

Criterio 4 — bajo esfuerzo matemático

Evita áreas muy matemáticas.

Trayectoria	Tiempo	Salario	Afinidad	Dificultad

4. ¿Dónde entra el LLM?

Esto es la parte más importante.

Te dicen:

“incorporando un modelo de lenguaje (LLM) como parte funcional del sistema”

Eso significa:

El LLM no puede ser decoración.

No sirve:

Sistema normal
+
ChatGPT explicando bonito

El LLM tiene que hacer una función real.

Y el PDF te lo dice explícitamente:

El sistema debe utilizar un LLM para analizar y comparar cualitativamente las trayectorias generadas.

Es decir:

El algoritmo genera trayectorias

(con búsqueda, grafos, restricciones, optimización)

El LLM las evalúa cualitativamente

Ejemplo:

Tu algoritmo produce:

A:

Frontend → Full Stack → Architect

B:

Data Analyst → Data Scientist

C:

Cybersecurity → Security Engineer

Luego el LLM analiza:

Trajectory A is ideal for users who prefer creativity,
UI work, and fast job entry.

Trajectory B offers stronger long-term salaries and
aligns better with mathematically inclined users,
but requires more sustained learning.

Trajectory C is highly stable and demand-driven,
though certification requirements may slow progress.

Eso es análisis cualitativo.

No numérico.

El LLM compara pros/contras, coherencia, riesgos, etc.
